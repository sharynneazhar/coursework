EECS 368 - Programming Languages & Paradigms

Table of Programming Paradigms

Paradigm Name Paradigm Description Parent(s)
Imperative This is a way of programming that explicitly describes the actions of a program. It uses a sequence of statements that directly modify a program state. N/A
Declarative These are programs that describe what they want to do but not how to get it. N/A
Object-Oriented A paradigm where data structures form objects that contain methods (i.e. functions) and attributes. They are created/instantiated from blueprints or classes. Imperative
Logic These programs are engines that answer questions based on facts and rules. They only care about what is allowed and what is not. Declarative
Structured These are task-centered programs. A structured program take statements derived from imperatives and declaratives which eliminates the GOTOs. Hence, giving us a nice block of code to read. Imperative, Declarative
Procedural Procedural programs are based on procedures (i.e. functions) which can be called from anywhere in the code. These are more about the order of the procedures as well as that states they are changing. Imperative, Structured
Functional These are identical to procedural programming except they do not contain states or variables. Declarative, Structured
